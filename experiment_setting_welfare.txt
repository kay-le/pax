Welfare-Maximising Opponent Shaping — Full Experiment Design
=============================================================

Goal: Train welfare-maximising meta-agents with IR constraints (Lagrangian dual ascent),
      compare against original Shaper/MFOS baselines from scalable opponent shaping.

Architecture:
  - WelfareShaperAtt: single GRU + self-attention (from Shaper), works in ALL environments
  - WelfareShaper: 3-stream GRU + theta gating (from MFOS), works in IPD only (not STORM)

Reference types for IR constraint:
  - IR calibration: opponent learns against fixed (random) shaper → "what you'd get without shaping"
  - Self-play: both agents learn independently with PPO → "what you'd get under naive learning"

Pipeline: Phase 0 (baselines) → Phase 1 (references) → Phase 2 (welfare training) → Phase 3 (evaluation)

=============================================================
COMPUTE NOTES
=============================================================
- Original CoinGame configs use num_devices=4, STORM configs use num_devices=8.
- For single-GPU: add ++num_devices=1. Effective popsize = popsize * num_devices,
  so for equivalent scale on 1 GPU, multiply popsize accordingly
  (e.g., STORM: popsize=1024 with num_devices=1 instead of popsize=128 with num_devices=8).
- Welfare runners (welfare_evo, welfare_rl) only support num_devices=1 currently.
  Multi-device support uses pmap in the rollout; welfare_evo already has this.
- IPD experiments are lightweight (minutes). CG/STORM are heavier (hours-days depending on GPU).
